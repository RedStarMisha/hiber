Работает обязательно с актуатором

Клиент при запуске регистрируется в Эврике. При штатном завершении работы он сам себя удаляет из нее.
Во время работы клиент должен отправлять в эврику информацию о том что он жив - обновлением. Если сервис перестает присылать
информацию по определенному периоду,Эврика начинает процесс удаления из реестра. Если все же сервис откликается с большим периодом
включается режим самосохранения(прекращения удаления).
то эврика удалит данный сервис из реестра


eureka.server.enable-self-preservation=true - включает отключает самосохранение
eureka.server.expected-client-renewal-interval-seconds=30 - период тактовых вызовов клиента
eureka.instance.lease-expiration-duration-in-seconds=90 - период в который сервер может удалить сервис из реестра
eureka.server.eviction-interval-timer-in-ms=60 запуск задачи по очистке реестра от неработающих сервисов с таким периодом
eureka.instance.lease-renewal-interval-in-seconds=30 - период клиента обращения на сервер Эврика

eureka.client.fetch-registry - указывает должен ли клиент стягивать информацию реестра Eureka из сервера
eureka.client.register-with-eureka - указывает должен ли клиент регистрироваться в Eureka
